# Phase 4, Wave 3: Sidebar UX & Polish

## Goal
Add a sidebar displaying project and model info, along with a button to re-index the ChromaDB corpus by invoking `scripts/ingest.py`.

<threat_model>
- Low: Application hangs when indexing large corpus -> Indexing happens synchronously in Streamlit, spinner alerts user. Button protects accidental clicks or multiple invokes.
</threat_model>

## Tasks

```xml
<task>
  <description>Create Sidebar Info Panel</description>
  <read_first>
    - app.py
  </read_first>
  <action>
    Add a sidebar using `with st.sidebar:`.
    Add title `st.title("Project Info")`.
    Add `st.markdown` displaying the model: "LLM: gemma3:4b, Embeddings: qwen3-embedding:0.6b".
    Display basic corpus info via `st.markdown`: "Data source: `documents/` directory".
  </action>
  <acceptance_criteria>
    - grep -q "st.sidebar" app.py
    - grep -q "gemma3:4b" app.py
  </acceptance_criteria>
</task>

<task>
  <description>Add Re-index Button</description>
  <read_first>
    - app.py
    - scripts/ingest.py
  </read_first>
  <action>
    In the sidebar, add a button `if st.button("Re-index corpus"):`.
    When clicked, use `with st.spinner("Re-indexing corpus..."):`.
    Inside the spinner context, use Python's `subprocess.run` to execute `python scripts/ingest.py` (ensure to capture stdout/stderr).
    After completion, trigger `st.success("Indexing complete!")`.
  </action>
  <acceptance_criteria>
    - grep -q "st.button(\"Re-index corpus\")" app.py
    - grep -q "subprocess.run" app.py
  </acceptance_criteria>
</task>
```

## Verification

- Start the app with `streamlit run app.py`.
- Click the "Re-index corpus" button in the sidebar.
- Verify the indexing command runs as a subprocess and the success message appears after indexing completes.
