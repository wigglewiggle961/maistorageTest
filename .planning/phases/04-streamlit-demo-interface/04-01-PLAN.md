# Phase 4, Wave 1: Streamlit Scaffold & Chat UI

## Goal
Set up the `app.py` Streamlit entrypoint with page configuration and the chat interface components, including session state for chat history.

<threat_model>
- High/High: Exposure of internal paths via stack traces -> handled by catching exceptions in Streamlit and displaying user-friendly error messages.
</threat_model>

## Tasks

```xml
<task>
  <description>Create initial app.py with page config</description>
  <read_first>
    - .planning/phases/04-streamlit-demo-interface/04-CONTEXT.md
    - .planning/phases/04-streamlit-demo-interface/04-UI-SPEC.md
  </read_first>
  <action>
    Create `app.py` in the project root.
    Add `import streamlit as st`.
    Call `st.set_page_config(page_title="Azure AI Foundry RAG", page_icon="🤖", layout="centered")`.
    Add a main title: `st.title("Azure AI Foundry Agent RAG")`.
  </action>
  <acceptance_criteria>
    - grep -q "st.set_page_config" app.py
    - grep -q "layout=\"centered\"" app.py
  </acceptance_criteria>
</task>

<task>
  <description>Implement chat history interface</description>
  <read_first>
    - app.py
  </read_first>
  <action>
    Initialize `st.session_state.messages` to an empty list if not present.
    Loop through `st.session_state.messages` and display them using `with st.chat_message(message["role"]): st.markdown(message["content"])`.
    Show an initial welcome message in the UI if the session state is empty (matching UI-SPEC empty state body): "Ask a question about the Microsoft Azure AI Foundry Agent Service. The agent will read the official docs and gather citations for you."
  </action>
  <acceptance_criteria>
    - grep -q "st.session_state.messages" app.py
    - grep -q "st.chat_message" app.py
  </acceptance_criteria>
</task>

<task>
  <description>Implement chat input handler</description>
  <read_first>
    - app.py
  </read_first>
  <action>
    Add a chat input: `if prompt := st.chat_input("Ask a question about the Microsoft Azure AI Foundry Agent Service..."):`
    When the user submits, append `{"role": "user", "content": prompt}` to `st.session_state.messages` and display it in a `st.chat_message("user")` block.
    For now, append a placeholder assistant message `{"role": "assistant", "content": "I am a placeholder agent response."}` and display it in a `st.chat_message("assistant")` block.
  </action>
  <acceptance_criteria>
    - grep -q "st.chat_input" app.py
  </acceptance_criteria>
</task>
```

## Verification

- `python -c "import app"` executes without errors.
- Run `streamlit run app.py` and submit a message, verify it appears in chat.
