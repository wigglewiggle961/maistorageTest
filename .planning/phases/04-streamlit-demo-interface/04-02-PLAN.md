# Phase 4, Wave 2: Agent Output Integration

## Goal
Integrate the Streamlit UI with the actual LangGraph agent to stream the reasoning steps and display the final answer with citations.

<threat_model>
- High/High: Hallucination injection -> agent hallucination check node in LangGraph flow handles it, but UI must faithfully display exact answer returned.
</threat_model>

## Tasks

```xml
<task>
  <description>Integrate LangGraph agent execution</description>
  <read_first>
    - app.py
  </read_first>
  <action>
    Import the agent runner from `src`. (Assuming `from src.agent import run_agent_workflow` or equivalent entrypoint built in Phase 3).
    Replace the placeholder assistant response logic with actual graph invocation. 
    Wrap the invocation in a `st.status("Evaluating document relevance...", expanded=False) as status:` to fulfill the English-status requirement.
  </action>
  <acceptance_criteria>
    - grep -q "st.status" app.py
  </acceptance_criteria>
</task>

<task>
  <description>Render Reasoning Steps</description>
  <read_first>
    - app.py
  </read_first>
  <action>
    As the graph executes, log the raw node names inside the `st.status` context using `st.write("Step: [node_name]")`. (This satisfies the dev-transparency requirement from the UI-SPEC).
    Once finished, update the status to `status.update(label="Answer generated", state="complete", expanded=False)`.
  </action>
  <acceptance_criteria>
    - grep -q "status.update" app.py
  </acceptance_criteria>
</task>

<task>
  <description>Display Citations and Answer</description>
  <read_first>
    - app.py
  </read_first>
  <action>
    Extract the final answer and sources from the agent's output state.
    Display the main answer using `st.markdown()`. 
    Create an expander below the answer: `with st.expander("Sources"):` and list the retrieved chunks with their file names and section headers to satisfy Citation UX.
    Append the fully formatted response dictionary (answer + sources markup) to `st.session_state.messages` to preserve it in history.
  </action>
  <acceptance_criteria>
    - grep -q "st.expander(\"Sources\")" app.py
  </acceptance_criteria>
</task>
```

## Verification

- Running `streamlit run app.py` connects to the graph and gives a real answer with a status container tracking steps.
