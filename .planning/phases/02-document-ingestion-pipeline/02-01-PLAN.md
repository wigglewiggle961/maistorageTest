# Phase 2: Document Ingestion Pipeline

## Goal
Parse, chunk, embed, and store all markdown documents and image summaries into a persistent ChromaDB vector store. Process FAQ pairs to build an evaluation dataset.

## Must Haves
- Python script `scripts/ingest.py` effectively triggers formatting and ingestion without duplicates.
- The `src/vectorstore.py` encapsulates `Chroma` for straightforward interaction in later phases.
- `eval_dataset.json` correctly populates with cross-verified questions and chunks.

## Tasks

<task>
  <description>Create the VectorStore Wrapper</description>
  <read_first>
    - src/config.py
  </read_first>
  <action>
    Create `src/vectorstore.py`. Implement a `ChromaWrapper` class parameterized with `CHROMA_DIR`, `CHROMA_COLLECTION` and using `langchain_ollama.OllamaEmbeddings` configured with `EMBED_MODEL` and `OLLAMA_BASE_URL` from `src.config`.
    Expose:
    - `add_documents(docs: list[Document]) -> None` (upserts into ChromaDB by generating chunk IDs via sha256 of `chunk.page_content + chunk.metadata['source']`).

    - `collection_stats() -> dict` returning subset counts.
  </action>

  </acceptance_criteria>

</task>

  <description>Implement Markdown Loader & Chunker</description>
  <read_first>
    - src/config.py
  </read_first>
  <action>
    Create `src/ingestion.py`. Add a function `load_and_chunk_markdown() -> list[Document]`.
    - Iterate over `*.md` files in `DOCUMENTS_DIR`.
    - Use `MarkdownHeaderTextSplitter` on `[("#", "h1"), ("##", "h2"), ("###", "h3"), ("####", "h4")]`.
    - Use `RecursiveCharacterTextSplitter(chunk_size=1024)` on resulting chunks that exceed size without breaking existing code blocks.
    - Set metadata: `source` (relative to project root), `section_header` (h1..h4 combined), and `chunk_index`.
  </action>
  <acceptance_criteria>
    - `src/ingestion.py` exists and contains `MarkdownHeaderTextSplitter`.
