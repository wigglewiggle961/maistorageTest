## 1. System Classification

**System Type:** Agentic RAG

**Description:**
A local-first Agentic Retrieval-Augmented Generation system that answers developer questions about Microsoft Azure AI Foundry Agent Service. It uses a stateful LangGraph orchestrator to conditionally route queries, retrieve Markdown documentation from ChromaDB, grade relevance using an LLM judge, generate grounded answers, check for hallucinations, and conditionally re-write queries to re-retrieve if grading fails.

**Critical Failure Modes:**
1. **Hallucination:** Generating responses that are not explicitly backed by the retrieved markdown chunks.
2. **Infinite Loops:** Failing to break out of the "re-write and re-retrieve" cycle when a satisfactory answer doesn't exist.
3. **Scope Bleed:** Answering questions outside the narrow scope of Azure AI Foundry Agents.

## 2. Framework Decision

**Selected Framework:** LangGraph

**Version:** >=0.0.30

**Rationale:**
LangGraph provides the exact mathematical graph abstraction needed to implement cyclic execution (re-retrieval loops) while maintaining a strict, observable `AgentState`. This is a hard requirement for Phase 3's "grade -> rewrite -> retrieve" cycle, preventing the black-box abstraction failures typical of standard LangChain zero-shot agents.

**Alternatives Considered:**

| Framework | Ruled Out Because |
|-----------|------------------|
| LlamaIndex Workflows | Less mature state orchestration; developer requested LangGraph explicitly in project requirements. |
| CrewAI | System is a single-agent RAG pipeline; multi-agent collaboration overhead is unnecessary. |

**Vendor Lock-In Accepted:** Yes, tied to the LangChain ecosystem.

## 3. Framework Quick Reference

> Fetched from official docs by `gsd-ai-researcher`. Distilled for this specific use case.

### Installation
```bash
pip install langgraph langchain-core langchain-community
```

### Core Imports
```python
from langgraph.graph import StateGraph, END
from typing import TypedDict, List
```

### Entry Point Pattern
```python
from typing import TypedDict, Annotated, Sequence
import operator

class AgentState(TypedDict):
    question: str
    generation: str
    documents: list[str]
    retries: int

workflow = StateGraph(AgentState)
workflow.add_node("retrieve", retrieve_node)
workflow.add_node("grade", grade_node)
workflow.add_node("generate", generate_node)
workflow.set_entry_point("retrieve")
workflow.add_edge("retrieve", "grade")
# Add conditional edges from grade based on state
app = workflow.compile()
```

### Key Abstractions
| Concept | What It Is | When You Use It |
|---------|-----------|-----------------|
| `StateGraph` | A state machine defining the flow. | To orchestrate the entire RAG pipeline. |
| `Node` | A python function receiving and returning state. | For individual steps (retrieve, grade, generate). |
| `Conditional Edge` | A routing function based on state. | For grade evaluation (if fail -> rewrite, else -> generate). |

### Common Pitfalls
1. **State Mutation:** Forgetting that state updates in LangGraph are shallow merges by default for dictionaries, or require explicit reducers (like `operator.add` for lists) if appending.
2. **Infinite Loops:** Failing to implement a `retries` counter in `AgentState`, causing conditional edges to loop indefinitely when grading consistently fails.

## 4. Implementation Guidance

**Model Configuration:**
- Primary LLM: `gemma4:e4b` via local Ollama.
- Temperature: `0.0` (for grading and strict retrieval to limit hallucinations).

**Core Pattern:**
Agentic RAG using a "Corrective RAG" (CRAG) style loop.

**Tool Use:**
No generic LLM `tools` (function calling). The "tools" are the explicit LangGraph nodes (retriever wrapper).

**State Management:**
Use a basic `TypedDict` containing: `question`, `documents` (retrieved chunks), `generation` (final answer), and `retries` (integer counter).

**Context Window Strategy:**
Pass Top-K retrieved documents from ChromaDB, truncated if they exceed token limits, prior to passing to the generation node.

## 4b. AI Systems Best Practices

> Written by `gsd-ai-researcher`. Cross-cutting patterns every developer building AI systems needs — independent of framework choice.

### Structured Outputs with Pydantic

Use Pydantic to ensure the LLM outputs strict JSON for routing and grading steps, avoiding regex parsing errors.

```python
from pydantic import BaseModel, Field

class GradeResult(BaseModel):
    is_relevant: bool = Field(..., description="True if the document is relevant to the question.")
```

### Async-First Design

While local Ollama may bottleneck on simultaneous inference, structure the graph nodes as asynchronous functions to allow for non-blocking I/O during ChromaDB retrieval.

### Prompt Engineering Discipline

- Separate the SYSTEM instruction from the USER query.
- Instruct the judge strictly: "You are a grader assessing if a document contains keywords or semantic meaning relevant to the user question. Return only a boolean."
