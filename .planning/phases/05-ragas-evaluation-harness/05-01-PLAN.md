# Plan 1: Naive Baseline Implementation

## Goal
Build a clean, non-agentic RAG baseline that performs a single retrieve → generate loop
(no routing, no relevance grading, no hallucination check, no re-retrieval). This baseline
uses the **same prompt and LLM** (`gemma4:e4b`) as the agentic flow to ensure a fair
comparison isolating the value of the LangGraph agentic nodes.

<action>
Create `src/baseline.py`. This module must expose a single public function:
`run_baseline(question: str) -> dict`

The function must:
1. Instantiate `ChromaWrapper` from `src.vectorstore`.
2. Call `similarity_search(query=question, k=config.TOP_K)` to retrieve the top-K chunks.
3. Format each retrieved `Document` into a string, using the SAME format as `retrieve_node`
   in `src/agent.py`:
   ```
   SOURCE: {doc.metadata.get("source", "unknown")}
   SECTION: {doc.metadata.get("section_header", "unknown")}
   CONTENT: {doc.page_content}
   ```
4. Instantiate `ChatOllama` with `model=config.LLM_MODEL`, `base_url=config.OLLAMA_BASE_URL`,
   `temperature=0`, and `keep_alive=-1`.
5. Invoke the LLM with the **exact same** system prompt and human prompt template as
   `generate_node` in `src/agent.py`:
   - System: "You are a technical assistant answering questions about Azure AI Foundry Agents.
     Use ONLY the provided context blocks to answer the question. If the answer is not in the
     context, say you don't know.\n\nStyle Requirement: Use inline citations like [1], [2]..."
   - Human: "Context:\n{context_str}\n\nQuestion: {question}"
6. Return a dict with these keys:
   - `"answer"`: the LLM's string response (`.content`)
   - `"contexts"`: a **list of strings** — each item is the full formatted doc string

The function must NOT contain any routing, grading, hallucination check, or retry logic.

Ensure `src/agent.py` exposes a clean entry point for the evaluation harness.
Add the following public function at the bottom of `src/agent.py` (after `compile_graph`):

```python
def run_agent(question: str) -> dict:
    """Public entry point for programmatic use by the evaluation harness.
    
    Returns:
        dict with keys:
            "answer": str - the final generated answer
            "contexts": list[str] - the formatted document strings used for generation
    """
    graph = compile_graph()
    result = graph.invoke({"question": question, "generation": "", "documents": [], "retries": 0})
    return {
        "answer": result.get("generation", ""),
        "contexts": result.get("documents", [])
    }
```
